# JLPT Master 前端页面设计文档

> 设计讨论稿 · 2026-04-21

---

## 整体框架

**导航结构：** 顶部固定导航栏，5个功能 Tab + 右侧全局设置

```
┌─────────────────────────────────────────────────────────────────┐
│ 日  JLPT Master │ 语料分析 │ JLPT专题 │ 实时视频 │ 知识库 │ 内化学习 │  N2 ▾  Gemini 2.5 Flash ▾ │
└─────────────────────────────────────────────────────────────────┘
```

**全局设置（TopBar 内嵌）：**
- 目标等级：N1 / N2 / N3 / N4 / N5（影响分析提示词中的难度预期）
- 模型选择：Gemini 2.5 Flash / Gemini 2.0 Flash / Gemini 2.5 Pro

**设计基调：**
- 底色白色，轻边框卡片
- 强调色：靛蓝 (#6366F1)
- 无暗色模式
- 无分析历史记录

## Tab 1 — 语料分析

**核心目的：** 输入日语文本或截图 → AI 流式分析所有句子 → 点击已完成句子查看词汇+语法 → 选择性入库 → 追问

### 输入方式

- **文本**：直接粘贴，支持多行
- **截图**：上传图片 → AI OCR 提取文本（可编辑确认）→ 同文本路径继续
  - OCR 完成后显示识别文本供用户确认/修正，再点"分析"
  - AI 分析时同时传入原图 + 确认文本，保留视觉上下文

### 整体布局

```
┌────────────────────────────────────────────────────────────────┐
│  [文本]  [截图]                                                  │  ← 输入模式 Tab
│                                                                 │
│  文本区域 / 图片上传 + OCR 结果文本框                             │
│                                               [分析]            │
└────────────────────────────────────────────────────────────────┘

┌─────────────────────────┬──────────────────────────────────────┐
│  分词分句               │  AI 分析卡片                          │
│                         │                                      │
│  ○ 昨日、学校へ…        │                                      │
│  ● 先生に会って…  ←选中 │  先生に会って、宿題を提出しました。   │
│  ● 帰り道で…            │  与老师见面，提交了作业。             │
│  ○ また明日…            │                                      │
│                         │  词汇 · 4                            │
│                         │  [先生] [宿題] [提出] …              │
│                         │                                      │
│                         │  语法 · 2                            │
│                         │  [〜て（连用）] [〜ました]            │
│                         │                                      │
└─────────────────────────┴──────────────────────────────────────┘
```

### 句子状态（左侧）

| 状态 | 视觉 | 可交互 |
|------|------|--------|
| 等待分析 | ○ 灰色文字 | 否 |
| 分析完成 | ● 正常文字，带标记 | 可点击 |
| 当前选中 | ● 高亮背景 | — |

**流式过程**：AI 逐句完成后，对应句子状态从"等待"变为"已完成"，用户即可点击。

### 右侧初始状态

未点击任何句子时，显示提示：`← 点击左侧已分析的句子查看详情`

### 词汇 Chip 展开态（右侧卡片内）

```
  ┌─────────────────────────────────────────┐
  │ 昨日  きのう  [N4]              ▲        │
  │ ─────────────────────────────────────   │
  │  意思：昨天                              │
  │  词性：名词・副词                         │
  │  例句：昨日は雨でした。                   │
  │                         [+ 加入知识库]   │
  └─────────────────────────────────────────┘
```

### 追问面板

分析完成后，右侧卡片底部出现追问入口（针对当前句子）：

```
  [对比] [用法] [活用] [例句] [自由追问]
  词语 ___                       [发送]
  ──────────────────────────────────────
  AI 回复
```

### 已确定的设计决策

- 语料分析只做**文本 + 截图**，不在此 Tab 放 JLPT 题型
- AI **一次性分析所有句子**（流式），不支持单句单独触发
- 右侧**单一卡片**，点击左侧句子切换内容
- 已分析完成的句子在左侧有视觉标记，未完成不可点击

## Tab 2 — JLPT 专题

**核心目的：** 针对 JLPT 固定题型的专项练习。与语料分析分开，因为题型有固定格式，未来会内置题库。

### 布局构成

```
  JLPT 专题
  针对考试题型的专项练习，未来将内置官方风格题库

  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
  │ 文字・語彙 │  │   文法    │  │   読解    │  │   聴解   │
  │ 汉字/读音 │  │ 语法选择  │  │ 短文阅读  │  │ 听力练习  │
  │ N5~N1    │  │ N5~N1    │  │ N3~N1    │  │ N3~N1   │
  │ [开发中] │  │ [开发中] │  │ [开发中] │  │ [开发中] │
  └──────────┘  └──────────┘  └──────────┘  └──────────┘

  内置题库功能开发中，敬请期待
```

### 待讨论问题

- 这个 Tab 的第一版是纯占位，还是希望先做一个"自由出题"功能（让 AI 根据等级出一道题）？
- 出题是基于知识库中已有的词汇/语法，还是完全由 AI 自由生成？

## Tab 3 — 实时视频

**核心目的：** 视频字幕 → 分词展示 → 点词查字典 → 选句 AI 分析（低 AI 消耗的大量输入流）

### 布局构成

```
[ 字幕加载区 ]
  ┌─────────────────────────────────────────────────┐
  │ 🔗  https://youtube.com/watch?v=...    [加载字幕] │
  └─────────────────────────────────────────────────┘

[ 字幕列表 · 加载后出现 ]
  字幕 · 124 句    [全选]  [分析选中句 (3)]

  ┌──────────────────────────────────────────────────┐
  │ □  00:12  [日本語][の][勉強][は][楽しい][です]     │
  │ ☑  00:24  [毎日][練習][する][こと][が][大切][です] │  ← 已选中，高亮
  │ □  00:38  [先生][に][教え][て][もらい][ました]     │
  └──────────────────────────────────────────────────┘
              ↑ 点击任意 token → 弹出字典查询浮窗

[ 字典浮窗 ]
  ┌──────────────────┐
  │  練習            │
  │  れんしゅう       │
  │  练习，训练       │
  │  [N4]            │
  └──────────────────┘
```

### 技术依赖

- 分词：调用后端 `/api/preprocess`（本地 Janome，无 AI 消耗）
- 字典：调用后端 `/api/dictionary/{word}`
- 字幕获取：需要新后端端点 `/api/video/subtitles`（YouTube/Bilibili 字幕下载，后端尚未实现）

### 待讨论问题

- 字幕加载目前依赖一个尚未实现的后端端点。第一版是否先支持**手动粘贴字幕文本**，作为过渡方案？
- 点击"分析选中句"后，是在当前页面内展示分析结果，还是跳转到语料分析 Tab？
- 分词 token 的粒度问题：Janome 日语分词颗粒度较细，"行きました"会被切成"行き"+"まし"+"た"，显示效果是否需要做合并？

## Tab 4 — 知识库

**核心目的：** 浏览、搜索所有积累的词汇和语法原子（Atom）。统计概览 + 表格列表 + 详情页。

### 布局构成（列表页）

```
  知识库
  你积累的每个词汇和语法点

  ┌──────┐  ┌──────┐  ┌──────┐
  │  47  │  │  31  │  │  16  │
  │ 全部  │  │ 词汇  │  │ 语法  │   ← 统计卡片
  └──────┘  └──────┘  └──────┘

  [🔍 搜索词条…]  [全部] [词汇] [语法]   ← 筛选工具栏

  ┌──────────────────────────────────────────────────┐
  │  词条     类型   成熟度          属性数            │
  ├──────────────────────────────────────────────────┤
  │  昨日     词汇   ████░░  82      4               │
  │  〜ました  语法   ██░░░░  45      3               │
  └──────────────────────────────────────────────────┘

  共 47 条 · 第 1/2 页    ← →
```

### 布局构成（详情页）

```
  ← 返回知识库

  昨日        [词汇]  入库 2026-04-20   [删除]
  ────────────────────────────────────────────

  属性 · 4
  ┌ AI 提取 ──────────────────────────────┐
  │  reading     きのう                    │
  │  meaning     昨天                     │
  │  part_of_speech  名词・副词            │
  │  example     昨日は雨でした。          │
  └────────────────────────────────────────┘

  关联 · 1
  ─ synonym → 昨日 [词汇]

  来源分析 · 2
  ─ 自由文本   2026-04-20
  ─ 自由文本   2026-04-19
```

### 待讨论问题

- "成熟度"目前是一个后端字段，含义是什么？是否值得在界面上展示，还是太抽象？
- 是否需要在详情页支持**手动添加属性**（用户自己补充释义、例句）？
- 来源分析列表——有无必要显示？用户场景是否真的会反查"这个词从哪段文章学到的"？

## Tab 5 — 内化学习

**核心目的：** 主动回忆，把知识库中的积累真正内化。提供多种练习模式。

### 布局构成

```
  内化学习
  选择今天要练习的方式

  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
  │ 翻訳练习  │  │ 書き取り  │  │ 比較学习  │  │ 听解练习  │
  │ 日→中理解 │  │ 听写填空  │  │ 近义辨析  │  │ 配合视频  │
  │  [可用]  │  │ [开发中] │  │  [可用]  │  │ [开发中] │
  └──────────┘  └──────────┘  └──────────┘  └──────────┘

  练习来源：[知识库全部 ▾]   等级筛选：[N2 ▾]
```

### 翻訳练习 Session

```
  第 3 题 / 47                              [重新开始]

  ┌──────────────────────────────────────────┐
  │                                          │
  │         練習する                          │  ← 当前词汇
  │         请翻译成中文                       │
  │                                          │
  │  ┌────────────────────────────────────┐  │
  │  │  你的翻译…                          │  │
  │  └────────────────────────────────────┘  │
  │                                          │
  │                      [跳过]  [查看答案]  │
  └──────────────────────────────────────────┘

  ── 点击查看答案后 ──

  参考释义：练习，训练
  ┌─────────────────────────────────────────┐
  │  ✓ 记住了      △ 模糊      ✗ 不会       │  ← 自我评估
  └─────────────────────────────────────────┘
```

### 比較学习 Session（构想）

```
  以下两个词有什么区别？

  見る  vs  見える

  [ AI 解释对比 ]
  ─────────────────────────────────────────────
  見る：主动动作，"看"
  見える：自然进入视野，"看得见/看到了"
  …
```

### 待讨论问题

- 翻訳练习的评分机制：是纯粹的用户自评（记住/模糊/不会），还是用 AI 判断翻译的正确性？
- 练习来源如何选择？是固定用全部知识库，还是需要让用户选"最近加入的 20 个"、"N2 词汇"等维度？
- 比較学习的词对从哪里来？是用户手动输入两个词，还是从知识库中自动推荐近义词对？
- 书写练习（書き取り）的交互设计：是让用户手写（需要画布），还是键盘输入假名？

## 跨页面交互

- **语料分析 → 知识库：** 入库后点击"已加入"直接跳转到 Atom 详情页
- **实时视频 → 语料分析：** 选中句子后跳转到语料分析 Tab，文本预填充
- **内化学习 → 知识库：** 练习中可能需要查看某词的完整属性
- **TopBar 设置 → 语料分析：** 目标等级影响 AI 分析的输出侧重

## 尚未确定的设计决策

| 决策点 | 选项 A | 选项 B | 备注 |
|--------|--------|--------|------|
| 截图输入入口 | 分析类型子 Tab 中加"截图"选项 | 单独的上传按钮 | 当前后端支持 image 字段 |
| 视频字幕获取 | 先做手动粘贴文本 | 等待后端字幕端点 | 后端尚未实现 |
| 实时视频分析结果 | 在当前页面内展示 | 跳转到语料分析 Tab | 影响页面复杂度 |
| 内化学习评分 | 纯用户自评 | AI 判断 + 用户确认 | AI 判断增加消耗 |
| 内化学习词对来源 | 用户手动输入 | 从知识库自动推荐 | 需要语义相似度功能 |
| 知识库成熟度显示 | 保留进度条 | 隐藏，太抽象 | 看用户心智模型 |

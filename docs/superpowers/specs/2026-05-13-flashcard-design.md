# 内化学习 — 卡牌记忆功能设计文档

*创建日期：2026-05-13*

---

## 一、功能定位

内化学习模块的第一个功能。用户从知识库中提取原子，通过卡牌划拨的方式做主动召回练习，结果写入 Trace 驱动后续复习优先级。

核心理念：正面给提示 → 大脑先检索 → 翻转对比 → 左划不会 / 右划会 → 立即记录轨迹。

---

## 二、整体数据流

```
用户进入内化学习页
        ↓
选择设置（数量 N、提示属性类型、可选标签过滤）
        ↓
GET /api/internalize/queue?limit=N&prompt=reading&tag=N2
        ↓
后端按优先级分数排序返回 N 个原子
优先级 = ↑历史失败率 + ↑距上次复习天数（实时计算，不持久化）
        ↓
前端持有完整卡堆（内存，会话期间不再请求）
        ↓
用户每次划卡（左=不会 / 右=会）
→ 立即 POST /api/internalize/trace { atom_id, result: 'know'|'unknown' }
→ 写入 traces 表  action='review'  detail={ result, prompt_type }
        ↓
所有卡划完 → 显示本轮结果汇总（会 X 张 / 不会 Y 张）
```

**不新增任何数据库表**，复用现有 `traces` 表。

---

## 三、后端 API

### 3.1 获取复习队列

```
GET /api/internalize/queue
Query: limit(int), prompt(string), tag(string, optional)
```

返回按优先级排序的原子列表。每个原子包含：
- 基本信息（id、type、key）
- JLPT 等级：从 `atom_tags` 中读取（tag 值为 'N1'~'N5'，无则视为 N5）
- 指定的提示属性值（prompt_value）
- 背面所有属性列表（properties）

优先级分数计算（SQL 层）：
```
score = (失败次数 / max(总复习次数, 1)) * 0.6
      + (1 - exp(-天数/14)) * 0.4
```
天数 = 距上次复习的天数，14 天衰减常数。新原子（从未复习）天数按 999 计，优先级最高。

### 3.2 记录划卡结果

```
POST /api/internalize/trace
Body: { atom_id: UUID, result: 'know' | 'unknown', prompt_type: string }
```

写入 `traces` 表：`action='review'`，`detail={ result, prompt_type }`。

## 四、卡牌视觉设计

### 4.1 等级风格系统

卡牌视觉由两个维度叠加：**JLPT 等级**（稀有度）× **原子类型**（配色）。

| 等级 | 风格 | 边框 | 光效 |
|------|------|------|------|
| N5 | 白纸/入门 | 细线，无装饰 | 无 |
| N4 | 羊皮纸/木质 | 木纹纹理，暖棕色 | 无 |
| N3 | 青铜/石质 | 铜色浮雕边框 | 轻微金属光泽 |
| N2 | 白银/冰晶 | 银色流光边框 | 蓝白扫光动画（循环） |
| N1 | 黄金/传说 | 金色粒子边框 | 持续粒子流 + 光晕 |

原子类型配色：
- **词汇**：蓝色系底色
- **语法**：橙色系底色

### 4.2 卡牌正面结构

```
┌─────────────────────────────┐
│  [词汇 / 语法]      N2 ●   │
│                             │
│                             │
│        提示属性大字          │
│                             │
│                             │
│    ─────────────────────    │
│        点击翻转查看          │
└─────────────────────────────┘
```

背面：日语写法 + 读音 + 释义 + 例句，属性按类型分组折叠显示。

## 五、动画系统

### 5.1 会话开始

1. **洗牌**：3-4 张卡快速交错扇开再收回，模拟实体洗牌（200ms）
2. **出牌**：当前卡从牌堆弹出，spring 过冲后落定

### 5.2 卡堆层次（同时渲染 3 张）

```
[当前卡]          ← 可交互，全尺寸
  [第2张]         ← scale 0.95，translateY +8px
    [第3张]       ← scale 0.90，translateY +16px，透明度 60%
```

### 5.3 拖拽交互

- 跟随手势实时旋转（最大 ±15°）
- 左侧：红色遮罩渐显 + "不会" 大字
- 右侧：绿色遮罩渐显 + "会" 大字
- 阈值：拖拽超过卡宽 30% 触发决策

### 5.4 划出动画

| 场景 | 动画 |
|------|------|
| 右划（会） | 卡片加速飞出屏幕右侧，轻微旋转 |
| 左划（不会） | 飞出前短促震动（x: 0→-8→0→-5→0），再飞出左侧 |
| 未过阈值释放 | Spring 弹回原位 |
| 下一张卡 | scale 0.95→1.0 + translateY 弹起 |

### 5.5 翻转

Y 轴 3D 翻转（rotateY 0°→90°，切换内容，90°→180°），duration 400ms，ease-in-out。

### 5.6 等级专属出牌动画

| 等级 | 出牌效果 |
|------|---------|
| N5 / N4 | 普通 spring 弹入 |
| N3 | 弹入 + 轻微震动落定 |
| N2 | 弹入后边框扫光一次 |
| N1 | 屏幕压暗 → 中心金色光环扩散 → 卡牌从上方落下（scale 1.2→0.95→1.0）→ 粒子边框持续流动 |

### 5.7 会话结束

牌堆消失 → 结果卡从下方弹入 → "会 N 张 / 不会 M 张" 数字递增动画。

## 六、练习设置（会话开始前）

用户可配置：
- **数量**：10 / 20 / 50 / 自定义
- **提示属性**：中文释义 / 读音 / 例句（选其一作为正面提示）
- **标签过滤**（可选）：按 JLPT 等级或自定义标签筛选范围

---

## 七、技术选型

- **手势与动画**：Framer Motion（`useMotionValue` + `useTransform` 实现拖拽联动，`AnimatePresence` 管理卡片进出）
- **粒子效果（N1）**：CSS `@keyframes` 或轻量 canvas，不引入重量级粒子库
- **后端**：FastAPI 新增 `/api/internalize/` 路由，复用现有 `traces` 表

---

## 八、范围约束

本期不包含：
- SRS 间隔重复算法（二期可升级）
- 音频 TTS 播放
- 离线缓存
- 多用户

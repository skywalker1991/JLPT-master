## 一、产品哲学

### 背单词是低效的

传统语言学习依赖词汇表和孤立语法条目的死记硬背。这种方式的根本缺陷不是"记不住"，而是**记住了也不会用**——大脑存储的是没有激活路径的孤立节点，遇到真实场景时无法自然调用。

### 语言是网络，不是列表

真正的语言能力来自于对**语言思维模式**的理解：为什么日本人这么说？这个表达和那个表达有什么区别？什么场景用正式，什么场景用口语？这些理解无法靠背诵获得，只能通过大量情境化输入和主动分析来建立。

### 为什么是现在

大模型在语言领域的能力是所有领域中最强的：理解语境和语感、解释语法背后的逻辑、对比近义表达的微妙差异、评价造句是否地道、用自然语言回答"为什么"。

传统语言学习工具的瓶颈在于——理解语言需要智能，而过去的软件没有智能。Anki 只能翻转卡片，不能告诉你"为什么用ので而不是から"；Duolingo 只能判断对错，不能解释"语法对但日本人不会这么说"。这个产品在 AI 时代才真正可行，因为它的核心能力——分析、提炼、评价、解释——恰好是大模型最擅长的事。

### AI 说的，并不是你内化的

```
AI 告诉你「結局」表示无奈      → 信息
你在三次不同场景中感受到这种无奈  → 知识
你能在自己说话时自然用出来       → 内化
```

信息、知识、内化——三者之间有不可跳过的鸿沟。AI 只能提供信息，但信息不等于你的知识，更不等于你能用出来。**这个系统要解决的，就是从 AI 的信息到人的内化之间的鸿沟。**

因此：
- AI 负责提取信息，但人自己判断"我会不会"（元认知是内化的一部分）
- 知识必须通过主动输出（造句、拼写、听写）来检验，而不是只看 AI 的分析
- 内化需要时间和重复，所以系统记录行为轨迹，驱动有针对性的复习

## 二、创始人学习经验

- 英语：雅思 7.5
- 日语：JLPT N2

**有效的方法：**
从短句和真实场景出发，同时习得语法与词汇，注重阅读和听力输入，从中总结语法规律、词汇用法与语言思维模式。

**遇到的瓶颈：**
提炼出的知识散落在 AI 对话中，难以复习、搜索和量化；知识条目彼此孤立，缺乏对比与关联；没有系统的主动输出练习。

**方法论验证：**
这套学习方法与语言习得研究高度吻合——Krashen 的可理解输入假说、Schmidt 的注意假说、Nation 的附带习得理论。两门语言（英语雅思7.5、日语N2）的实际成绩在不同语种上复现了同一套方法，证明其有效性不是偶然。

**为什么这套方法有效——认知科学基础：**

这套方法暗合大脑学习语言的底层机制，不是偶然有效：

1. **深加工优于浅加工**（Craik & Lockhart）：从句子和场景出发，同时处理意思、语境、语感、语法结构，加工深度远超"看字形背翻译"的单词表
2. **网络记忆优于孤立记忆**：一个知识点连接的其他知识越多，被激活的入口越多，越难遗忘
3. **编码-提取情境匹配**（Tulving 编码特异性）：从真实内容中学习，将来在真实场景中使用，编码和提取的情境天然一致
4. **元认知促进记忆**：自己判断"会不会"迫使大脑做一次检索尝试，失败的检索本身就加强后续记忆
5. **区分性学习**：追问"为什么用这个不用那个"，建立近义表达的边界，这在高级阶段（N2、N1）尤为关键

**适用范围：**
这套方法适用于所有阶段的学习者。方法论本身不变，变的只是输入内容的难度——初学者的输入是简单短句和基础场景，进阶者的输入是新闻长文和考试真题。AI 提取、人筛选、入库建关联、输出练习，这个流程对所有级别通用。

## 三、核心学习模型

### 两个阶段

```
阶段一：输入与提炼
真实内容 → AI 分析 → 人工筛选 → 知识库

阶段二：内化与输出
知识库 → 多种练习形式 → 真正习得
```

### 完整闭环

```
真实内容输入
（文章 / 截图 / JLPT 题目 / 听力）
        ↓
本地预处理（即时）+ AI 分析（流式）
        ↓
探索路径：用户沿着内容追问
  每一步追问都可能产出新的原子和关系
  "这个和那个有什么区别？" → 对比关系
  "这个还有什么用法？" → 新属性
        ↓
人工筛选 → 知识库
（带上下文、带关联的知识单元）
        ↓
多种输出练习
（翻译造句 / 拼写 / 听力 / 对比练习）
        ↓
知识图谱可视化
（所有知识节点与关联关系的网络图）
```

- 粘贴文本
- 截图（OCR 识别文字）
- 视频字幕流（YouTube、日剧、电影、直播等）：观看过程中实时逐句分析词汇和语法，用户边看边判断入库

### JLPT 题型专属分析模板

| 题型 | AI 输出内容 |
|------|-----------|

| **排序题** | 正确排序方案 + 排序依据的语法知识 |
| **阅读题** | 逐句翻译 + 词汇/语法注释 + 选项整体分析 |
| **听力题** | 词汇/语法解析 + 口语表现专项分析 |
| **自由文本** | 分句 + 词汇提取 + 语法识别 + 语言思维注释 |

## 五、知识库设计

### 设计原则

> **原子只创建一次，但终身生长。每次重逢都是一次加深理解的机会。**

### 概念模型

知识库的核心模型由三个概念构成。这三个概念是知识表示的通用结构，不依赖具体语种——任何语言的知识都可以用「概念是什么 + 概念的特征 + 概念之间的联系」来表达。不同语言的差异只体现在属性类型和关系类型的扩展上，核心模型不变。

#### 原子（Atom）

系统中最小的不可再分的知识单元。只有两种：

- **词汇原子**：能在字典中查到的最小意义单位。包括名词、动词、形容词、副词、动词短语（如「気にする」）、语气词（如「ね」「よ」）等。
- **语法原子**：句子的组合规则或句型。包括接续规则、助词用法、敬语形式等。

每个原子在系统中**只存在一个实例**，通过唯一标识去重。

原子的识别与匹配策略因类型而异：
- **词汇原子**：通过形态素分析还原到辞书形，可确定性匹配
- **语法原子**：语法没有统一的"辞书形"，不同教材对同一语法的表记不同，因此通过语义匹配来识别，最终由用户确认是否为同一原子

#### 属性（Property）

描述原子某一方面特征的信息片段。

- 属性不是固定结构，而是**开放集合**——可以随时追加新的属性类型
- 每条属性记录**来源**和**时间**，反映"什么时候、从哪里学到的"
- 同一个原子可以拥有同一类型的多条属性，不覆盖，只追加
- 属性类型示例：读音、释义、语体（书面/口语/正式/随意）、口语变形、使用条件、语感、语言思维笔记、用户造句等

#### 关系（Relation）

两个原子之间的有意义连接。

- 关系有**类型**，起步阶段收敛为以下核心类型：
  - 近义对比（から↔ので）
  - 书面↔口语（やはり↔やっぱり）
  - 同根变形（〜てしまう↔〜ちゃう）
  - 使用条件差异（〜うちに↔〜あいだに）
  - 语感/情绪差异（結局↔最終的に）
- 类型集合未来可扩展，但起步时保持克制，避免关系噪音
- 关系可以携带**说明**（为什么建立这个关联、区别是什么）

### 关于语言思维与口语表现

这两者不是独立的原子类型：

- **语言思维**（如"日语用被动态表达困扰"）是跨越多个原子的宏观认知，作为**笔记类属性**挂在相关原子上，一条笔记可以关联多个原子
- **口语表现**（如缩约、省略、语气词）是词汇和语法原子的**属性**（口语变形），而语气词本身是独立的词汇原子

### 知识库就是你自己的词典

通用词典（如 JMdict）是所有人共用的、静态的。用户的知识库是从真实探索中生长出来的**个人词典**——每个条目带着你的语境、你的理解、你的学习轨迹。AI 和用户在探索过程中，不断丰富这本属于自己的词典。它比任何通用词典都更贴合你的学习阶段和认知方式。

### 知识生长模型

原子不是一次录入完成的，而是随着学习不断丰满：

```
第1次遇到 → 创建种子（基本信息：写法、读音、释义）
第2次遇到 → 补充属性（语体、使用条件、语感）
第3次遇到 → 建立关系（发现近义词、对比词）
第N次遇到 → 追加笔记、新例句、新发现
```

每次遇到已有的原子，系统**识别出已存在**，引导补充而非重复创建。

原子的**成熟度**（属性数量、关系数量、复习次数）反映学习深度，最终通过知识图谱的节点大小可视化呈现。

### 核心约束

1. **原子唯一**：每个知识点只创建一次，终身不重复
2. **属性追加**：知识只增不覆盖，每次重逢是补充而非修改
3. **关系生长**：原子之间的连接随学习深入而逐渐变密
4. **来源可溯**：每一条属性和关系都记录从哪里、什么时候获得
5. **成熟可见**：原子的丰满程度反映学习深度

### 应用层概念

以上三个概念是知识模型的核心，描述的是"知识本身"——概念是什么（原子）、概念的特征（属性）、概念之间的联系（关系）。删掉任何一个，知识模型就不完整。

系统在应用层还需要其他概念来支撑功能，这些概念不属于知识模型，可以随系统功能扩展而不断增加：

- **标签（Tag）**：原子的分组标记，用于主题归类和筛选
- **轨迹（Trace）**：用户与原子的交互行为记录（入库、练习、复习），驱动复习优先级和成熟度计算
- **分析记录（Analysis）**：用户每次输入的原始内容和 AI 分析结果，是未经清洗的原始数据
- **专题集、练习会话、内置语料、用户配置**等，按需增加

### 知识供给模型

> **AI 负责提取和呈现，人负责判断和取舍，系统在入库时发现关联。**

知识从 AI 分析到进入知识库，经过三个环节，各有明确分工：

#### 第一步：AI 提取（自动）

AI 分析用户输入的内容（文章、截图、JLPT 题目等），输出结构化的词汇和语法列表。AI 只负责识别和呈现，不判断用户"会不会"。

#### 第二步：人工筛选（人）

用户浏览 AI 输出的列表，凭自己的判断决定：
- 这个我不会 → 选择加入知识库
- 这个我会 → 跳过

"判断自己会不会"本身就是学习的一部分（元认知），这个判断不应由系统代劳。

#### 第三步：入库时关系发现（系统）

用户选择入库的那一刻，系统检索已有知识库，发现相似原子并提示：

```
用户选择加入「ので」
      ↓
系统发现已有「から」，都表示原因
      ↓
提示："「から」已在库中，要建立对比关系吗？"
  [建立] [跳过]
```

关系发现的时机选在入库时，因为此刻用户的注意力正集中在这个具体原子上，建立关联的认知成本最低。

#### 再次遇到已有原子时

用户选择加入一个已存在的原子，系统识别并引导补充新属性（新的例句、语感、使用条件等），而非重复创建。

### 关联建立方式

- **入库时系统推荐**：加入新原子时，系统检索已有相似原子并推荐关联
- **用户确认**：一键确认或跳过，无需手动填写对比内容
- **手动建集合**：用户可创建专题集（如"N2必考语法100个"、"容易混淆的副词"）

## 六、输出练习模块

### 1. 翻译造句练习

- 系统提供母语（中文）句子
- 用户输入日语翻译
- AI 评分并分析错误原因：哪里语法错、哪里表达不地道、正确形式是什么

题目来源：从知识库条目**反向自动生成**对应的母语练习句。

### 2. 单词拼写练习

- 看中文 → 输入日语（假名或汉字）
- 听 TTS 音频 → 输入日语
- 三种模式：假名→汉字 / 汉字→假名 / 中文→日语

### 3. TTS 听力模拟

- 系统用 TTS 朗读知识库中的例句或自定义文本
- 用户听写或回答问题
- 可调速：0.75x 精听 / 1.0x 正常 / 1.25x 挑战
- 专项分析口语表现（省略、缩读、语气词）

### 4. 对比练习（进阶）

- 从已建立的关联关系自动生成"选择正确用法"题
- 直接针对容易混淆的语法对或词汇对
- 这是区别于 Anki 的核心差异化功能

## 七、知识图谱可视化

> 图谱是系统的终态，不是起点。优先级低于输入分析、知识库和输出练习。先做完最小学习闭环，再建图谱。

图谱的价值不在于展示，而在于**驱动练习**——从关联关系自动生成对比练习题，精准打击容易混淆的知识点。如果图谱不能参与出题，它就只是装饰。

系统最终呈现一张**属于用户自己的语言知识网络**：

- **节点**：每个学过的词汇或语法点
- **边**：节点之间建立的关联关系
- **节点颜色**：词汇（蓝）/ 语法（橙）
- **节点大小**：原子成熟度越高（属性越多、关系越密、复习越多），节点越大
- **边的颜色**：近义（红）/ 正式-口语（紫）/ 同根变形（灰）
- **点击节点**：查看详情、例句、来源题目
- **过滤**：按 JLPT 级别、主题标签筛选

### 这张网能告诉你什么

| 观察 | 含义 |
|------|------|
| 孤立节点（无边） | 学了但未理解其在语言中的位置 |
| 密集节点群 | 在该领域理解深入 |
| 两点之间无连线 | 知道但未建立对比意识 |
| 整体网络密度 | 语言知识体系的完整程度 |

知识图谱不是装饰，而是**语言理解深度的真实映射**。

## 八、目标用户

**自我驱动的语言学习者。**

这个系统不推课程、不设每日任务、不做游戏化激励。它假设用户自己知道要学什么、主动带着内容来——一篇文章、一道题目、一段听力。系统的价值是帮你把这些内容转化为可沉淀、可关联、可复习的个人知识。

具体包括：
- 日语个人自学者
- JLPT 备考学习者（N5 ~ N1）

系统会内置 JLPT 各级别的考试内容（真题、语法、词汇等）作为现成的学习素材。用户可以用自己找到的内容，也可以直接使用系统提供的考试内容，走的是同一套学习流程。

### 多语言扩展性

本系统的核心概念模型（原子、属性、关系、例句、标签、轨迹）不依赖具体语种。语言扩展路线：

1. **日语**（当前）— 以 JLPT 备考为切入点
2. **英语**（下一步）— 以雅思/托福等考试为切入点

不同语言需要适配的是输入分析模块（题型模板、分词方式）和语言特有的属性类型（如日语的敬语体系、英语的时态体系），但底层的知识生长模型是通用的。

## 九、与现有产品的差异

| 产品 | 做了什么 | 停在哪里 |
|------|---------|---------|
| Duolingo | 游戏化练习，降低入门门槛 | 停在信息层：告诉你对错，不解释为什么 |
| Anki | SRS 间隔重复，对抗遗忘 | 停在信息层：孤立卡片，无上下文，无关联 |
| Language Reactor | 影视沉浸，大量真实输入 | 停在信息层：看过即忘，无结构化沉淀 |
| ChatGPT 对话 | AI 分析解释，质量高 | 停在信息层：分析完散落在对话中，无法复习 |
| **本产品** | AI 提取 → 人筛选 → 知识库生长 → 输出练习 | **跨越信息到内化的鸿沟** |

核心差异：其他产品都停在"信息"这一步。本产品解决的是从 AI 的信息到人的内化之间的完整路径——提取、筛选、关联、输出、复习。

## 十、实现优先级

### MVP（最小学习闭环）

先做能完整走通"输入→入库→练习"的最小版本：

1. **输入分析**：粘贴文本 / 截图 → AI 提取词汇和语法列表
2. **人工筛选入库**：用户选择不会的原子 → 入库时发现关联
3. **输出练习**：翻译造句（AI 评分）+ 对比选择题（核心差异化）
4. **基础复习**：列表形式，基于轨迹调整优先级

### 第二阶段

- JLPT 题型专属分析模板
